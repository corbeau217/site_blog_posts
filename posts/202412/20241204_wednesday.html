  • animations system

    • original idea is that there would be some function of x,y,z, that would create points in model space
    • following on from that was the idea we could use a function of a transformation matrix, to change it over some number of frames
    • these steps would then either be used as the locations for the model to move to on each frame of the animation
    • the design of the interface would be very similar to the current shapes interface
    • having sliders for precision/point count/scale
    • checkboxes for whether it animates in a loop, whether the camera orbits, or if the model/path is shown

    • the different views would be

      • 1. scene view - shows the points/path/model/grid lines/axises
      • 2. controls - is where the sliders and checkboxes would go to manipulate the behaviour of the scene
      • 3. values/scene graph hierarchy - would show locations for each of the frames, and or translation/rotation/scale of the model at those points
      • 4. menu/settings - things like whether we show the grid lines/axises/colour code the axises as RGB or just single colour
